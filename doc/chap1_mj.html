1.1-1 CategoryConstructor

The function takes no arguments but various options:

  • category_object_filter

  • commutative_ring

  • list_of_operations_to_install

  • is_monoidal

  • create_func_list

  • create_func_object_or_fail

  • create_func_other_object

  • create_func_other_morphism

1.2 Example

This is an example of a category which is created using CategoryConstructor out of no input. This category "lies" in all doctrines and can hence be used (in conjunction with LazyCategory) in order to check the type-correctness of the various derived methods provided by CAP or any CAP-based package.
